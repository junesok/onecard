# 원카드 — 기획서

엑셀 스프레드시트로 위장한 실시간 멀티플레이어 원카드 게임.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| 장르 | 실시간 멀티플레이어 카드 게임 |
| 인원 | 2~5명 |
| UI | Microsoft Excel 스프레드시트 위장 |
| 통신 | Socket.io (포트 3000) |
| 이모지 | 조커 카드에만 허용, 나머지 UI 전면 금지 |

---

## 2. 카드 구성

표준 52장 + 조커 2장 = **총 54장**

| 종류 | 무늬 | 랭크 |
|------|------|------|
| 일반 카드 | ♠ ♥ ♦ ♣ | A, 2~10, J, Q, K |
| 흑백 조커 | — | 🃏 (흑백) |
| 컬러 조커 | — | 🃏 (컬러) |

### 무늬 색상
- **빨강**: ♥ 하트, ♦ 다이아
- **검정**: ♠ 스페이드, ♣ 클로버

---

## 3. 게임 흐름

### 준비
1. 닉네임 입력 → 대기실 입장
2. 방장이 2명 이상 모이면 시작 가능
3. 게임 시작 → 각자 7장 배분 (비공개)
4. 덱에서 1장 공개 → 필드 카드로 설정
5. 시작 플레이어 랜덤 결정 → 시계 방향으로 진행

### 차례 진행
1. 패에서 낼 수 있는 카드 선택 후 제출
   - 낼 카드 없으면 덱에서 1장 드로우 → 드로우 후 즉시 턴 종료
2. 특수 카드 효과 처리
3. 다음 플레이어로 턴 이동

### 승리
- 가장 먼저 손패를 모두 비운 플레이어 승리
- 덱이 소진되면 필드 카드를 제외한 버린 패를 섞어 새 덱으로 사용

---

## 4. 카드 내기 규칙

### 단일 카드
- 필드 카드와 **같은 무늬** 또는 **같은 숫자(랭크)** 일 때 낼 수 있음

### 여러 장 동시에 내기
- **같은 숫자** 카드만 여러 장 동시에 낼 수 있음
- 같은 무늬 여러 장 동시 불가
- 마지막에 놓인 카드가 새 필드 카드

---

## 5. 특수 카드 효과

| 카드 | 효과 | 방어 가능 카드 |
|------|------|---------------|
| A (스페이드 제외) | 다음 플레이어 +3 드로우 | 아무 A / 같은 무늬 2 / 색깔 맞는 조커 |
| 2 | 다음 플레이어 +2 드로우 | 아무 2 / 같은 무늬 A |
| 흑백 조커 | 다음 플레이어 +5 드로우 (스페이드·클로버 필드에서만 사용) | 스페이드 A / 컬러 조커 |
| 스페이드 A | 다음 플레이어 +5 드로우 | 흑백 조커만 |
| 컬러 조커 | 다음 플레이어 +5 드로우 (하트·다이아 필드에서만 사용) | 방어 불가 |
| J | 다음 플레이어 스킵 (N장 → N명 스킵) | — |
| Q | 진행 방향 반전 (N장 → N회 반전) | — |
| K | 낸 플레이어가 한 번 더 (같은 무늬 또는 다른 K) | — |

### 페널티 누적
드로우 카드(A, 2, 조커류) 방어 시 페널티가 누적된다.  
방어하지 못한 플레이어가 누적 합산 장수를 드로우한다.

---

## 6. 원카드 선언

- 손패가 1장이 되면 해당 플레이어는 **미선언 상태**가 됨
- 미선언 상태는 게임 중 **언제든지** 스페이스바로 처리 가능
- **미선언 플레이어가 스페이스바** → 원카드 선언 성공, 패널티 없음
- **다른 플레이어가 스페이스바** → 미선언 플레이어 신고, 해당 플레이어 +1 드로우
- 이미 선언된 플레이어를 신고해도 무효

## 8. 카드 렌더링

### 카드 외형
- 흰 배경, 모서리 둥글게
- 좌상단·우하단에 랭크 + 무늬 기호 표시
- 빨강 카드(♥ ♦): 빨간 글씨
- 검정 카드(♠ ♣): 검정 글씨
- 뒷면: 단색 패턴 (파란 격자)

### 조커
- 흑백 조커: 🃏 + "JOKER" 텍스트, 흑백 배경
- 컬러 조커: 🃏 + "JOKER" 텍스트, 컬러 배경

### 카드 선택 (내 패)
- 클릭 시 카드가 위로 올라와 선택 표시
- 같은 숫자의 카드만 다중 선택 가능
- 선택 후 [내기] 버튼 또는 Enter로 제출

### 상대 패
- 카드 뒷면만 표시, 장 수 숫자로 표기
- 원카드 미선언 상태 플레이어는 이름 옆에 강조 표시

---

## 9. 조작

| 입력 | 동작 |
|------|------|
| 카드 클릭 | 선택 / 선택 해제 |
| [내기] 버튼 | 선택한 카드를 필드에 제출 |
| [드로우] 버튼 | 카드 1장 드로우 + 턴 종료 / 페널티 체인 중에는 누적 장수 드로우 |
| [턴 종료] 버튼 | 카드를 낸 뒤 명시적으로 턴 넘기기 (K 추가 턴 포기 포함) |
| [정렬] 버튼 | 손패를 무늬 → 숫자 순으로 정렬 |
| Space | 원카드 선언 (1장 남은 본인) 또는 신고 (다른 플레이어) |
| `` ` `` (백틱) | 보스키 (엑셀 화면 전환) |

### 버튼 활성화 조건

| 버튼 | 활성화 조건 |
|------|------------|
| 내기 | 내 턴 + 카드 1장 이상 선택됨 + 유효한 조합 |
| 드로우 | 내 턴 + 아직 드로우하지 않음 |
| 턴 종료 | 내 턴 + 카드를 1장 이상 낸 뒤 |
| 정렬 | 항상 활성 |

---

## 10. 채팅

- 게임 중 전체 공개 채팅 (모든 플레이어)
- 게임 종료 후에도 채팅 유지
- 우측 패널 하단에 배치

---

## 11. HUD 정보

- 현재 턴 플레이어 (강조 표시)
- 진행 방향 (시계 / 반시계)
- 덱 남은 카드 수
- 현재 누적 페널티 장 수 (0이면 비표시)
- 플레이어별 손패 장 수 + 원카드 선언 여부

---

## 12. Socket 이벤트 설계

| 이벤트 | 방향 | 설명 |
|--------|------|------|
| `join` | C→S | 닉네임과 함께 입장 |
| `start_game` | C→S | 방장이 게임 시작 |
| `game_state` | S→C | 초기 패 + 필드 카드 + 턴 정보 |
| `play_cards` | C→S | 카드 제출 (카드 ID 배열) |
| `draw_card` | C→S | 덱에서 드로우 요청 |
| `card_update` | S→C | 필드 카드 + 손패 변경 + 턴 정보 갱신 |
| `penalty_draw` | S→C | 페널티 누적 드로우 처리 결과 |
| `one_card_press` | C→S | 스페이스바 입력 |
| `one_card_result` | S→C | 원카드 선언/신고 결과 |
| `game_over` | S→C | 게임 종료 + 순위 |
| `chat` | C↔S | 채팅 메시지 |
| `rematch_vote` | C→S | 다시하기 투표 |
| `lobby_update` | S→C | 대기실 플레이어 목록 갱신 |

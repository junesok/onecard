# 원카드 — 기획서

엑셀 스프레드시트로 위장한 실시간 멀티플레이어 원카드 게임.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| 장르 | 실시간 멀티플레이어 카드 게임 |
| 인원 | 방당 2~5명, 동시 다중 방 운영 |
| UI | Microsoft Excel 스프레드시트 위장 |
| 통신 | Socket.io (포트 3000) |

---

## 2. 카드 구성

표준 52장 + 조커 2장 = **총 54장**

| 종류 | 무늬 | 랭크 |
|------|------|------|
| 일반 카드 | ♠ ♥ ♦ ♣ | A, 2~10, J, Q, K |
| 흑백 조커 | — | BW |
| 컬러 조커 | — | COLOR |

### 무늬 색상
- **빨강**: ♥ 하트, ♦ 다이아
- **검정**: ♠ 스페이드, ♣ 클로버

---

## 3. 게임 흐름

### 접속 및 방 입장
1. 닉네임 입력
2. 방 목록에서 방 선택 (공개/비공개) 또는 새 방 생성
3. 방장이 2명 이상 모이면 게임 시작 가능

### 게임 준비
1. 게임 시작 → 각자 7장 배분 (비공개)
2. 덱에서 1장 공개 → 필드 카드로 설정 (일반 숫자 카드 우선)
3. 시작 플레이어 랜덤 결정 → 시계 방향으로 진행

### 차례 진행
1. 패에서 낼 수 있는 카드 선택 후 제출
   - 낼 카드 없으면 덱에서 1장 드로우 → 드로우 후 즉시 턴 종료
2. 특수 카드 효과 처리
3. 다음 플레이어로 턴 이동

### 종료 (순위제)
- 패를 다 낸 플레이어부터 순위가 매겨지며, 완주한 플레이어는 관전 모드로 전환
- 마지막까지 남은 1명이 꼴찌 → 게임 종료
- 덱이 소진되면 필드 카드를 제외한 버린 패를 섞어 새 덱으로 사용

### 게임 종료 후
- 순위표 표시
- **다시하기 투표**: 전원 동의 시 같은 방에서 재시작 (로비로 복귀)
- **나가기**: 방 목록으로 복귀

---

## 4. 방 시스템

### 방 생성
- 방 이름 직접 입력 (기본값: `{닉네임}의 방`)
- 공개 / 비공개(비밀번호 설정) 선택

### 방 목록
- 로비 상태 또는 종료 상태인 방만 표시
- 게임 진행 중인 방은 목록에서 숨김
- 비공개 방은 `[비공개]` 표시

### 방 입장 제한
- 게임 진행 중인 방 입장 불가
- 최대 5명 초과 시 입장 불가
- 비공개 방: 비밀번호 불일치 시 입장 불가

---

## 5. 카드 내기 규칙

### 단일 카드
- 필드 카드와 **같은 무늬** 또는 **같은 숫자(랭크)** 일 때 낼 수 있음

### 여러 장 동시에 내기
- **같은 숫자** 카드만 여러 장 동시에 낼 수 있음
- 조커는 1장만 낼 수 있음
- 마지막에 놓인 카드가 새 필드 카드

### 조커 내기 조건
- **흑백 조커**: 검정 무늬(♠ ♣) 필드 또는 K 추가턴(검정 무늬) 시 사용 가능
- **컬러 조커**: 빨강 무늬(♥ ♦) 필드 또는 K 추가턴(빨강 무늬) 시 사용 가능

---

## 6. 특수 카드 효과

| 카드 | 효과 | 방어 가능 카드 |
|------|------|---------------|
| A (스페이드 제외) | 다음 플레이어 +3 드로우 | 아무 A / 같은 무늬 2 / 색깔 맞는 조커 |
| 스페이드 A | 다음 플레이어 +5 드로우 | 흑백 조커만 |
| 2 | 다음 플레이어 +2 드로우 | 아무 2 / 같은 무늬 A / 색깔 맞는 조커 |
| 흑백 조커 | 다음 플레이어 +5 드로우 | 스페이드 A / 컬러 조커 |
| 컬러 조커 | 다음 플레이어 +10 드로우 | 방어 불가 |
| 7 | 무늬 선언 (이후 해당 무늬 또는 7만 낼 수 있음) | — |
| J | 다음 플레이어 스킵 (N장 → N명 스킵) | — |
| Q | 진행 방향 반전 (N장 → N회 반전) | — |
| K | 낸 플레이어가 한 번 더 (같은 무늬 또는 다른 K) | — |

### 페널티 누적
드로우 카드(A, 2, 조커류) 방어 시 페널티가 누적된다.
방어하지 못한 플레이어가 누적 합산 장수를 드로우한다.

---

## 7. 원카드 선언

- 손패가 1장이 되면 화면 중앙에 `[X] 키를 눌러주세요` 배너가 표시됨
- 동시에 채팅창에도 안내 메시지 출력
- **1장인 플레이어가 먼저 키 입력** → 원카드 선언 성공, 패널티 없음
- **다른 플레이어가 먼저 키 입력** → 신고 성공, 1장인 플레이어 +1 드로우
- 대상 플레이어가 카드를 드로우하거나 게임 상태가 변경되면 배너 자동 소멸
- 채팅 입력창에 포커스 중일 때는 키 입력이 선언에 적용되지 않음

---

## 8. 관전 모드

- 완주한 플레이어는 자동으로 관전 모드로 전환
- 관전 중에는 남은 모든 플레이어의 패를 볼 수 있음 (상대방 카드 앞면 표시)
- 카드 내기 및 드로우 불가
- 채팅은 계속 가능

---

## 9. UI 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│ [Excel 타이틀 바]                                        │
├─────────────────────────────────────────────────────────┤
│ [리본 메뉴: 홈 / 삽입 / 페이지 레이아웃 / ...]           │
├─────────────────────────────────────────────────────────┤
│ [수식 입력창]  fx  =WAIT("플레이어명")                   │
├───────────────────────────────────┬─────────────────────┤
│                                   │ HUD 패널            │
│  [상대2] [상대3] [상대4] [상대5]  │ - 현재 턴 플레이어  │
│  (카드 뒷면 + 장 수)              │ - 진행 방향         │
│                                   │ - 덱 남은 카드 수   │
│       [덱]  [필드 카드]           │ - 누적 페널티       │
│                                   ├─────────────────────┤
│  [원카드 배너 / 결과 토스트]      │ 플레이어 목록       │
│                                   │ (장 수 + 원카드)    │
│   [내 카드] [내 카드] ...         ├─────────────────────┤
│   (앞면, 클릭하여 선택)           │ 채팅                │
│   [내기] [드로우] [정렬]          │                     │
└───────────────────────────────────┴─────────────────────┤
│ [Sheet1] [Sheet2]                                        │
└─────────────────────────────────────────────────────────┘
```

### ESC 키 (보스키)
- `ESC` → VSCode 에디터 화면으로 전환
- 다시 `ESC` → 게임 화면으로 복귀

## 10. 조작

| 입력 | 동작 |
|------|------|
| 카드 클릭 | 선택 / 선택 해제 |
| [내기] 버튼 | 선택한 카드를 필드에 제출 |
| [드로우] 버튼 | 카드 1장 드로우 + 턴 종료 / 페널티 체인 중에는 누적 장수 드로우 |
| [정렬] 버튼 | 손패를 무늬 → 숫자 순으로 정렬 |
| 랜덤 키 (배너 표시 시) | 원카드 선언 또는 신고 |
| `ESC` | 보스키 (VSCode 화면 전환) |

## 11. Socket 이벤트

| 이벤트 | 방향 | 설명 |
|--------|------|------|
| `join` | C→S | 닉네임 등록 |
| `join_ok` | S→C | 닉네임 등록 완료, 방 목록으로 이동 |
| `get_rooms` | C→S | 방 목록 요청 |
| `room_list` | S→C | 방 목록 전송 |
| `create_room` | C→S | 방 생성 (이름, 비밀번호) |
| `join_room` | C→S | 방 입장 (방 ID, 비밀번호) |
| `room_joined` | S→C | 방 입장 성공 |
| `room_error` | S→C | 방 입장 실패 (비밀번호 오류, 방 없음 등) |
| `leave_room` | C→S | 방 나가기 |
| `left_room` | S→C | 방 나가기 완료, 방 목록으로 이동 |
| `lobby_update` | S→C | 대기실 플레이어 목록 갱신 |
| `start_game` | C→S | 방장이 게임 시작 |
| `game_started` | S→C | 게임 시작 알림 |
| `game_update` | S→C | 전체 게임 상태 갱신 |
| `play_cards` | C→S | 카드 제출 (카드 ID 배열) |
| `draw_card` | C→S | 덱에서 드로우 요청 |
| `choose_suit` | C→S | 7 낸 후 무늬 선택 |
| `play_error` | S→C | 카드 제출 오류 |
| `one_card_challenge` | S→C | 원카드 발생, 랜덤 키 안내 |
| `one_card_key` | C→S | 원카드 키 입력 |
| `one_card_result` | S→C | 원카드 선언/신고 결과 |
| `game_over` | S→C | 게임 종료 + 순위 |
| `rematch_vote` | C→S | 다시하기 투표 |
| `rematch_status` | S→C | 투표 현황 |
| `chat` | C→S | 채팅 메시지 전송 |
| `chat_msg` | S→C | 채팅 메시지 수신 |
| `player_left` | S→C | 플레이어 나감 알림 |

## 12. 기술 스택

| 항목 | 기술 |
|------|------|
| Server | Node.js + Express + Socket.io |
| Client | Vanilla JS + HTML + CSS (단일 파일) |
| 렌더링 | HTML + CSS 카드 (Canvas 미사용) |
| 통신 | Socket.io 이벤트 기반 실시간 동기화 |
| 포트 | 3000 |
